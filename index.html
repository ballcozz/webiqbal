Soal


Diketahui:

Posisi head berada di track: 1234

[x1] adalah tanggal lahir masing-masing x 10, misal : 1 menjadi 10, 29 menjadi 290
[x2] adalah bulan lahir masing-masing x 100, misal : 4 menjadi 400, 11 menjadi 1100
[x3] adalah 4 digit tahun lahir masing-masing, misal : 2007, 2010
[x4] adalah 3 digit terakhir dari NIM, misal 395, 056 menjadi 56

Lakukan perhitungan banyaknya proses lintas track yang dilakukan menggunakan ketujuh algoritma (mulai dari PTPD hingga Circular Scan), dan gambarkan grafik perjalanan lintasan yang dilakukan serta narasi sebagai penjelasannya.

SSTF (Shortest Seek Time First)


Algoritma ini memilih track terdekat dari posisi head saat ini.

Start di 1234
Queue : [1600, 1050, 1775, 1950, 500, 10, 1100, 1000, 1900, 2004, 70, 80]

Proses :
1. Dari 1234 → terdekat : 1100 (jarak 134)
2. 1100 → 1050 (50)
3. 1050 → 1000 (50)
4. 1000 → 500 (500)
5. 500 → 1600 (1100)
6. 1600 → 1775 (175)
7. 1775 → 1900 (125)
8. 1900 → 1950 (50)
9. 1950 → 2004 (54)
10. 2004 → 80 (1924)
11. 80 → 70 (10)
12 → 10 (60)

Total = 134 + 50 + 50 + 500 + 1100 + 175 + 125 + 50 + 54 + 1924 + 10 + 60 = 4232

SCAN


Head bergerak naik ke atas sampai ujung, lalu balik turun.

Start: 1234, arah ke atas.
Track limit: 0 – 1999
Urutan naik: [1600, 1775, 1900, 1950, 2004]
Lalu balik: [1100, 1050, 1000, 500, 80, 70, 10]

Jalur:
- 1234 → 1600 → 1775 → 1900 → 1950 → 2004
- 2004 → balik ke 1100 → ... → 10

Total = (1600−1234) + (1775−1600) + (1900−1775) + (1950−1900) + (2004−1950) + (2004−1100) + (1100−1050) + (1050−1000) + (1000−500) + (500−80) + (80−70) + (70−10)

= 366 + 175 + 125 + 50 + 54 + 904 + 50 + 50 + 500 + 420 + 10 + 60 = 2764

C-SCAN


Head hanya bergerak satu arah (naik), lalu kembali ke bawah ekstrem tanpa melayani permintaan.

Naik: 1234 → 1600 → 1775 → 1900 → 1950 → 2004
Lalu: head lompat ke track 0 → naik: 10 → 70 → 80 → 500 → 1000 → 1050 → 1100

Total: (1600−1234) + (1775−1600) + (1900−1775) + (1950−1900) + (2004−1950) + (2004−0) [lompat] + (10−0) + (70−10) + (80−70) + (500−80) + (1000−500) + (1050−1000) + (1100−1050)

= 366 + 175 + 125 + 50 + 54 + 2004 + 10 + 60 + 10 + 420 + 500 + 50 + 50 = 3874

LOOK


Seperti SCAN, tapi hanya sampai permintaan terakhir (tidak sampai ujung disk)

Naik: 1234 → 1600 → 1775 → 1900 → 1950 → 2004 Turun: 1100 → 1050 → 1000 → 500 → 80 → 70 → 10

Lintasan sama seperti SCAN, jadi hasil sama: 2764

C-LOOK


Seperti C-SCAN, tapi lompat ke permintaan terkecil, bukan ke ujung disk.

Naik: 1234 → 1600 → 1775 → 1900 → 1950 → 2004 Lompat ke 10, lalu naik ke 70, 80, 500, 1000, 1050, 1100

Jarak: = 366 + 175 + 125 + 50 + 54 + (2004−10) [lompat] + 60 + 10 + 420 + 500 + 50 + 50 = 3860

PTPD


Karena ini tidak baku, kita anggap sebagai versi dinamis dari SSTF:
Setiap kali ada permintaan baru, algoritma mengevaluasi ulang head movement dan bisa mengganti jalur.

Kita asumsikan hasilnya mirip atau lebih baik dari SSTF.
Jadi hasilnya akan ≤ 4232 — kita pakai nilai ~4000 sebagai estimasi.
